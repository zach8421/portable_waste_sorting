# I8 — Advanced DBs: Which Bin API

> IMT 542 — I8 Assignment · Spring 2026 · Zach Greenman

A small Flask API that puts Seattle's "Where Does It Go?" sorting
guidance behind HTTP endpoints, serving the same portable JSON
structure produced by the notebook at the repo root.

This builds directly on the I7 individual API and the group project's
portable waste-sorting information structure.

## Data source

- **File:** [`../data/items.json`](../data/items.json) (310 items)
- **Type:** static JSON file used as a NoSQL-style document store
- **Origin:** Seattle Public Utilities — *Where Does It Go?* taxonomy,
  generated by [`../portable_waste_sorting.ipynb`](../portable_waste_sorting.ipynb)

The file is loaded once at server startup. An `item_id → item` map is
built in memory so single-item lookups are O(1) — the same access
pattern a small NoSQL key/value store would use.

## Install & run

```bash
cd i8
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python which_bin_api.py
```

Server listens on `http://127.0.0.1:5000`.

If port 5000 is taken on macOS (AirPlay Receiver uses it), disable
AirPlay Receiver in System Settings or change the `port=` argument
at the bottom of `which_bin_api.py`.

## Endpoints

| Method | Path                              | Purpose |
|--------|-----------------------------------|---------|
| GET    | `/`                               | API info, data source, list of endpoints |
| GET    | `/items`                          | All items (summary view) |
| GET    | `/items/<item_id>`                | One item (full record), 404 if missing |
| GET    | `/search?q=&stream=&category=`    | Filter by name/synonym, stream, and/or category (AND'd) |
| GET    | `/streams`                        | Counts of items per disposal stream |
| GET    | `/streams/<stream>`               | Items with a given disposal stream |
| GET    | `/categories`                     | Top-level categories with item counts |
| GET    | `/categories/<name>`              | Items in a top-level category |
| GET    | `/stats`                          | Summary stats (totals, stream counts, top categories) |

Valid disposal streams: `garbage`, `recycling`, `compost`, `donate`,
`dropoff`, `hazardous`, `special`.

## Example calls

```bash
# 1. API info
curl http://127.0.0.1:5000/

# 2. Look up one item by ID
curl http://127.0.0.1:5000/items/x130614

# 3. Missing item -> 404 JSON
curl -i http://127.0.0.1:5000/items/does-not-exist

# 4. Search by keyword + stream
curl 'http://127.0.0.1:5000/search?q=cardboard&stream=compost'

# 5. Search by category
curl 'http://127.0.0.1:5000/search?category=Plastic'

# 6. Bad query -> 400 JSON
curl -i 'http://127.0.0.1:5000/search'
curl -i 'http://127.0.0.1:5000/search?stream=notreal'

# 7. Everything that can be composted
curl http://127.0.0.1:5000/streams/compost

# 8. Category roll-up
curl http://127.0.0.1:5000/categories
curl http://127.0.0.1:5000/categories/Plastic

# 9. Summary stats
curl http://127.0.0.1:5000/stats
```

You can also paste any of these URLs into a browser.

## Error handling

- Missing item → `404 { "error": "item not found", "item_id": "..." }`
- Unknown disposal stream on `/streams/<stream>` →
  `404 { "error": "invalid stream", "valid_streams": [...] }`
- Empty category match → `404 { "error": "category not found or empty" }`
- `/search` with no params → `400 { "error": "provide at least one of q, stream, or category" }`
- `/search` with bad `stream=` → `400 { "error": "invalid stream", ... }`
- Unknown route → `404 { "error": "route not found" }`

All responses are `application/json`.

## How this satisfies the assignment

- **Builds on the existing API** (I7) using the same domain data — the
  portable JSON structure generated by this repo's notebook.
- **Data is served from an information structure**, not hardcoded —
  a JSON document store (NoSQL-style) loaded once at server startup
  and indexed in memory by `item_id`.
- **Multiple useful endpoints** beyond a single lookup: list, get-by-ID,
  search/filter, group-by-stream, group-by-category, and stats.
- **Realistic sample data** — 310 real Seattle Public Utilities items
  with disposal streams, conditions, synonyms, and category paths.
